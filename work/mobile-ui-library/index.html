Mobile UI Library & Design System

Unified tokens, components, and documentation so two mobile squads could ship consistent native experiences without bottlenecks.

Role: Product lead Timeline: 4 months Teams: iOS + Android + Design
Design Systems Mobile Product
Work together Back to work

Situation

Two mobile teams were shipping quickly but design drift and duplicated components slowed review cycles.

  • Inconsistent spacing, typography, and motion between iOS and Android
  • Design files and code weren’t linked, creating rework during QA
  • New contributors lacked guidance on patterns and accessibility

Approach

Led a lightweight system that matched how the teams actually built features.

  • Audited existing screens and created tokens for color, typography, and spacing
  • Shipped 40+ components in Figma with pairing guidelines to SwiftUI/Compose
  • Documented usage, props, and anti-patterns in a single reference site
  • Integrated accessibility checklists per component

Toolkit

  • Figma libraries + tokens
  • SwiftUI & Jetpack Compose examples
  • Design review checklist and linters
  • Async documentation + Loom walkthroughs

Outcomes

  • Cut design review churn by 35% by aligning tokens + components
  • New feature tickets referenced ready-made patterns, reducing build time
  • Design, iOS, and Android shipped the same UI without last-minute fixes
