Compose & RxJava Upgrades

Planned a safe migration path to Jetpack Compose and modernized reactive patterns while protecting delivery speed.

Role: Product + delivery Timeline: 8 weeks Scope: 3 mobile squads
Mobile Architecture Delivery
Work together Back to work

Situation

Android teams wanted Compose benefits but feared regressions and velocity loss.

  • Legacy XML views mixed with early Compose experiments
  • RxJava patterns varied widely between modules
  • Deadline pressure for new features continued

Approach

Sequenced the upgrade so teams could adopt Compose safely and incrementally.

  • Dependency audit and target versions with compatibility notes
  • Created migration playbooks per module (low, medium, high complexity)
  • Paired engineers on first two feature ports to prove the pattern
  • Added linters and regression checks to catch layout issues early

Collaboration

  • Weekly guild to share wins, blockers, and snippets
  • Documented dos/don’ts for Compose interop with existing views
  • Feature freeze windows scheduled with product for risky modules
  • Change log and rollback plan for every release

Outcomes

  • First Compose screens shipped without slowing feature delivery
  • Common Rx patterns reduced defects tied to threading
  • Clear upgrade plan reduced debate and kept leadership aligned
